# 🌐 Ecommerce Website

This website is a practice online shop called Cybermart. It acts like a real ecommerce store, but it is a demo/training project. The order/payment is simulated (for learning), not a real payment system like Amazon.
Things you can do on the website is:
1. You can look at products on the home page. <br>
2. You can click a product to see more details. <br>
3. You can add products to your cart. <br>
4. You can create an account or log in. <br>
5. You can go to checkout, enter shipping and payment info, and place an order. <br>
6. After placing an order, it shows a success message like a real store. 

# ⚙️ Technologies

This project was built with the following main technologies:
- React
- Vite
- React Router DOM
- React Hook Form
- Javascript (ES Modules + JSX)
- ESLint

# 🎈 The Process

### My Development Process

I built this project by focusing on the user experience first, then implementing each feature in small, manageable steps.

#### 1. Define the goal
My goal was to create a beginner-friendly ecommerce experience where a user can:
- browse products
- view product details
- add items to cart
- sign up or log in
- complete checkout
- see an order confirmation

#### 2. Map the user journey
Before writing code, I planned the full customer flow from landing on the homepage to placing an order.  
This helped me stay focused on what the user needs at each step.

#### 3. Build feature by feature
I broke the project into smaller parts so I could build and test each one clearly:
- navigation and page routing
- product list and product details
- cart management (add, remove, update quantity)
- authentication flow (signup/login/logout)
- checkout form and order confirmation

#### 4. Manage shared data carefully
I treated app data as three main groups:
- user authentication data
- shopping cart data
- checkout form data

Then I connected each page to the right data so the experience stayed consistent across the app.

#### 5. Improve usability
After the main flow worked, I focused on making the app easier to use:
- added validation with clear error messages
- improved cart controls and feedback
- supported saved address behavior
- made the layout responsive for desktop and mobile

#### What I learned
This project taught me how to think like both a user and a developer: plan the experience first, implement in small steps, and improve quality through iteration.
